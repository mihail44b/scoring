# Config-Driven Scoring Engine v2.0

Аналитическая платформа для оценки надежности и коммерческого потенциала B2B-контрагентов. 

Проект эволюционировал из простого скрипта в полноценную **Data-Driven** систему. Весь расчёт выполняется через независимый математический движок, который управляется динамическими JSON-пресетами. Это позволяет менять бизнес-логику (веса, пороги, справочники, правила) на лету через веб-интерфейс, без вмешательства разработчиков.

## 🚀 Ключевые возможности

- **Data-Driven Architecture**: Абстрактный движок (`core/engine.py`), который не привязан к конкретным колонкам, а интерпретирует правила из JSON-пресета.
- **Высокая производительность**: Алгоритмы оптимизированы для обработки больших Excel-файлов, внедрено O(1) кэширование колонок (`ColumnCache`).
- **Интерактивный дашборд**: Удобный веб-интерфейс для загрузки баз, визуальной аналитики сегментов («Горячие/Тёплые/Холодные»), регионов и качества данных.
- **Конфигуратор в реальном времени**: Встроенная панель настроек (`/settings`) со строгой валидацией через Pydantic. Позволяет управлять весами категорий, порогами логарифмирования и справочниками.

## 📊 Модель оценки (5 категорий)

Контрагенты оцениваются по 5 независимым векторам (веса настраиваются):
1. **Категория A (Финансовое здоровье)**: Выручка, прибыль, активы, закредитованность (с логарифмическим ранжированием и региональными коэффициентами).
2. **Категория B (Масштаб и зрелость)**: РМСП, количество сотрудников, срок на рынке.
3. **Категория C (Отраслевая релевантность)**: Оценка основного ОКВЭД и налогового режима по встроенным справочникам.
4. **Категория D (Доступность контакта)**: Наличие телефонов, email, сайта, ФИО руководителя.
5. **Категория E (Юридические риски)**: Маркеры ликвидации (жёсткий стоп-фактор), оценка ОКФС и ОКОПФ.

*(Полное описание математических моделей, формул и стоп-факторов доступно в файле [DOCUMENTATION.md](DOCUMENTATION.md))*

## 🛠 Как запустить у себя

**1. Установка зависимостей**  
Убедитесь, что установлен Python 3.10+. Откройте терминал в папке проекта и выполните:
```bash
pip install -r requirements.txt
```

**2. Запуск сервера**  
Запустите скрипт для локальной разработки:
```bash
python run_dev.py
```
*(Сервер запустится с включенным флагом `--reload` на порту 8000).*

**3. Использование**  
- **Дашборд скоринга**: Откройте браузер и перейдите по адресу [http://localhost:8000](http://localhost:8000). Загрузите ваш `.xlsx` файл для анализа.
- **Панель конфигуратора**: Доступна по адресу [http://localhost:8000/settings](http://localhost:8000/settings). Здесь вы можете менять логику скоринга на лету.
- **API Документация**: Интерактивный Swagger доступен по адресу [http://localhost:8000/docs](http://localhost:8000/docs).

## 💡 Особенности парсинга колонок
Движок ищет колонки в файле по **вхождению подстроки** без учета регистра (за это отвечает `ColumnCache`). Если в файле колонка называется `"Выручка (2025)"` или `"Выручка"`, система корректно её найдет. Привязки к конкретным форматам экспорта (СПАРК, Контур и др.) в коде нет, система универсальна.
