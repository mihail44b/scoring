# Скоринг Департамента Продаж (ДП)

Веб-приложение на FastAPI для автоматического скоринга по 5 категориям. Принимает на вход Excel-файл с сырыми данными, пропускает через скоринг-модели и отдает готовый файл с подсчитанными баллами и сегментацией.

## 🗂 Структура проекта

Архитектура спроектирована так, чтобы команда из 5 человек могла работать **параллельно** без merge-конфликтов. У каждого участника есть своя изолированная зона ответственности.

```text
dp_scoring/
├── app.py                      # Точка входа FastAPI, HTTP-эндпоинты
├── run_dev.py                  # Скрипт для удобного запуска сервера
├── requirements.txt            # Зависимости Python (pandas, fastapi, uvicorn и др.)
├── templates/
│   └── index.html              # Простой веб-интерфейс для загрузки файла
├── config/                     
│   ├── weights.json            # Веса категорий и пороги (Горячий/Теплый/Холодный)
│   └── *.json                  # Экспортированные справочники (ОКВЭД, ОКФС, ОКОПФ)
├── core/                       # Ядро системы (Зона Участника 5)
│   ├── validator.py            # Умная проверка входного файла на наличие нужных колонок
│   ├── feature_builder.py      # Оркестратор: последовательный вызов модулей A, B, C, D, E
│   ├── rules_engine.py         # Подсчет итогового балла и сегмента (с учетом конфига)
│   ├── result_builder.py       # Сборка финального DataFrame, переименование колонок на русский
│   └── exporter.py             # Конвертация DataFrame обратно в байты .xlsx
└── categories/                 # Модули бизнес-логики скоринга
    ├── category_a.py           # Фин. здоровье (Участник 1)
    ├── category_b.py           # Масштаб и зрелость (Участник 2)
    ├── category_c.py           # Отраслевая релевантность (Участник 3)
    ├── category_d.py           # Доступность контакта (Участник 4)
    └── category_e.py           # Юр. статус и риски (Участник 4)
```

## 🧑‍💻 Зоны ответственности и правила

- **Участники 1–4 (Аналитики)**: Пишут код *исключительно* внутри своих файлов в `categories/`. Их задача — принять `DataFrame`, реализовать формулы на Pandas (без привязки к конкретным годам в названиях колонок), добавить 3 своих колонки (`score`, `completeness`, `stop_factor`) и вернуть обновленный `DataFrame`.
- **Участник 5 (Архитектор/Бэкенд)**: Работает с папками `core/`, `config/` и файлом `app.py`. Следит за интеграцией, корректной выгрузкой, веб-интерфейсом и инфраструктурой.

## 🚀 Как запустить у себя

**1. Установка зависимостей**
Убедитесь, что у вас установлен Python (версии 3.9+). Откройте терминал в папке `dp_scoring` и выполните:
```bash
pip install -r requirements.txt
```

**2. Запуск сервера**
В этой же папке запустите скрипт для локальной разработки:
```bash
python run_dev.py
```
*(Сервер запустится с включенным флагом `--reload`, поэтому при изменении `.py` файлов он будет перезапускаться автоматически)*.

**3. Использование**
- **Графический интерфейс**: Откройте браузер и перейдите по адресу [http://localhost:8000](http://localhost:8000). Нажмите «Выберите файл», загрузите ваш `.xlsx` и получите результат.
- **Интерактивная API Документация**: Доступна по адресу [http://localhost:8000/docs](http://localhost:8000/docs). Через нее также можно вручную дергать API и тестировать эндпоинты.

## 📝 Замечания по входным данным
Валидатор и модули категорий ищут колонки по **вхождению подстроки** без учета регистра. Это означает, что если в сыром файле колонка называется `"Выручка (2025)"`, `"Выручка (2026)"` или просто `"Выручка"`, система корректно её найдет. Привязки к конкретным годам в коде нет!
